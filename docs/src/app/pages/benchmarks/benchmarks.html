Why These Numbers Matter

Nanosecond Resolution

At ~18ns per resolution, you can perform ~55 million service lookups per second per core. HTTP handlers typically resolve 3-5 services, adding only ~90ns overhead.

Lock-Free Design

Using DashMap for concurrent access means zero contention on reads. The 4-thread benchmark shows near-linear scaling with ~26µs per thread for 100 reads.

Efficient Scoping

Child scopes add only ~18ns when services are overridden locally, and ~36ns when resolving from parent. Perfect for request-scoped services.

Benchmark Methodology

Environment

  • Ubuntu Latest (GitHub Actions runner)
  • Rust 1.85+ stable toolchain
  • All features enabled (--all-features)
  • Release mode with LTO optimizations

Configuration

  • Criterion.rs 0.5 benchmarking framework
  • 100 samples per benchmark
  • 3-second warmup period
  • Statistical analysis with 95% confidence

What We Measure

  • Registration: Container + service setup
  • Resolution: Type lookup + factory call
  • Scoped: Parent chain traversal
  • Concurrent: Multi-thread scaling

Key Dependencies

  • DashMap 6: Lock-free concurrent hashmap
  • ahash: Fast TypeId hashing
  • once_cell: Lazy initialization
  • parking_lot: Optimized synchronization
